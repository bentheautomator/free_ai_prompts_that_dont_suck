# Free AI Prompts That Don't Suck — Project Rules

## What This Repo Is

A public collection of battle-tested system prompts for AI coding assistants. Every prompt solves a specific, documented failure mode. No vibes. No fluff.

## Structure

```
prompts/<category>/<slug>.md   — Source of truth. One file per prompt.
install/                       — AUTO-GENERATED by scripts/build.sh. Never hand-edit.
scripts/build.sh               — Generates install/ files and README table.
```

## Adding a New Prompt

1. Create `prompts/<category>/<slug>.md` with YAML frontmatter and the standard template
2. Run `make build` — generates install files and updates README table
3. Commit everything together (prompt + generated files)

## Rules

### Source of Truth
- `prompts/` is the only place prompt content is authored
- `install/` files are auto-generated — never edit them by hand
- README prompt table is auto-generated between `<!-- PROMPT_TABLE_START -->` and `<!-- PROMPT_TABLE_END -->`
- Run `make build` after any change to prompt files

### Prompt File Requirements
- YAML frontmatter: title, slug, category, tags, works_with, severity, one_liner
- slug must match filename (without .md)
- category must match parent directory name
- Instruction block must exist between two `---` horizontal rules in the body
- `make validate` catches all of these

### Categories
- New categories are just new directories under `prompts/`
- Build script discovers them automatically — no config to update
- Keep category names lowercase-kebab-case

### Commits
- `docs(prompts): add <name>` for new prompts
- One prompt per commit when possible
- Generated files go in the same commit as the source prompt
- Never commit install/ changes without corresponding prompts/ changes

## Tone
- Direct. No hedging.
- Written for practitioners.
- If it's true and it's funny, say it.
- Evidence over vibes. Every prompt traces back to a real failure.
